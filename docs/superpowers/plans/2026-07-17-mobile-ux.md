# JUNKRUN Mobile UX Refactor Implementation Plan (v2.3.0)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the UI into hub-and-spoke navigation with a glowing-card visual system so JUNKRUN feels like a mobile game — separation between sections, one job per screen — without touching the engine.

**Architecture:** `App` owns a two-level nav state (`activeTab` + `sub`); the MORE tab becomes a COMMAND hub of six tappable cards that open full-screen sub-screens, the MAP tab sheds Contracts/Signals into pill-button sub-screens, and the HUD gains a tap-toggled quick-stats drawer. Sub-screens render *inside* their tab component (early return), so tab components keep their local state (crucially MapScreen's in-flight route refs) while the sub is open. All new CSS lands in one task so later tasks touch only their own component files.

**Tech Stack:** Preact + @preact/signals, plain CSS in `src/style.css`, strict TypeScript, Vite single-file build. No new dependencies.

**Spec:** `docs/superpowers/specs/2026-07-17-mobile-ux-design.md` (committed). One normalization: the spec's sub-screen id `'settings'` is named `'options'` throughout (matches the OPTIONS card); the spec's "back slides out" exit animation is simplified to enter-only animations (slide-in on sub open, fade on tab/back) — exit animations would require dual-mounting old and new screens, which is not worth the complexity at this scale.

## Global Constraints

- **UI layer only.** Touch nothing outside `src/components/`, `src/app.tsx`, `src/style.css` (and `package.json` in U6). No changes to `src/engine/`, `src/config/`, save shape, or balance. Zero new save-compat surface.
- **Verbatim moves.** When this plan moves JSX/handlers/helpers between files, behavior must not change. The code blocks in this plan are the exact target content — use them verbatim.
- **Gates every task:** `npm run typecheck` (0 errors), `npm test` (87/87 — none of the 19 test files touch UI; any failure means you broke an import), `npm run build` (must succeed). Run all three before every commit.
- **reducedMotion:** all new animation must be CSS `animation`/`transition` (the existing `.reduced-motion *` global rule kills those). No JS-driven animation (`requestAnimationFrame` for motion) may be added.
- **Touch targets:** every new tappable is ≥ 44px min-height (`.hub-card`, `.pill-btn`, `.sub-header`, `.row-tap` provide this).
- **Theming:** new surfaces glow with the station accent via `var(--accent)`/`var(--glow)` using `color-mix(...)`, always preceded by a plain `rgba(...)` fallback declaration on the same property.
- **Keep the punch:** toasts, floaters, confetti, ticker, station theming, TradeSheet are untouched.
- Do not stage or commit the untracked `savegame` file or anything under `dist/`. `git add` only the files your task names. If `git commit` fails with `index.lock`, wait 2s and retry (max 5).

```tsx
import { useRef, useState } from 'preact/hooks';
import { store } from '../engine/store';
import { RELICS, relicCost, relicMaxLevel } from '../config/relics';
import { buyRelic, prestige, canPrestige, prestigePreviewDM } from '../engine/actions';
import { formatNum } from '../engine/num';

function PrestigeButton() {
  const [progress, setProgress] = useState(0);
  const rafRef = useRef<number | undefined>(undefined);
  const startRef = useRef(0);

  function start() {
    startRef.current = performance.now();
    const tick = (t: number) => {
      const p = Math.min(1, (t - startRef.current) / 3000);
      setProgress(p);
      if (p >= 1) {
        prestige();
        setProgress(0);
        return;
      }
      rafRef.current = requestAnimationFrame(tick);
    };
    rafRef.current = requestAnimationFrame(tick);
  }
  function cancel() {
    setProgress(0);
    if (rafRef.current) cancelAnimationFrame(rafRef.current);
  }

  return (
    <button
      class="btn btn-block btn-primary"
      style={{ position: 'relative', overflow: 'hidden' }}
      onMouseDown={start} onMouseUp={cancel} onMouseLeave={cancel}
      onTouchStart={start} onTouchEnd={cancel}
    >
      <div style={{ position: 'absolute', inset: 0, width: `${progress * 100}%`, background: 'rgba(255,255,255,0.3)', transition: progress === 0 ? 'width .15s' : 'none' }} />
      <span style={{ position: 'relative' }}>{progress > 0 ? 'HOLD…' : 'HOLD TO COLLAPSE 🌀'}</span>
    </button>
  );
}

export function WormholeScreen() {
  const s = store.value;
  const dmPreview = prestigePreviewDM(s);

  return (
    <>
      <div class="prestige-banner">
        <div>Collapse this run into Dark Matter.</div>
        <div class="dm-preview mono">+{formatNum(dmPreview)} 💠</div>
        <div style={{ fontSize: 11, opacity: 0.7, marginBottom: 10 }}>You have {formatNum(s.darkMatter)} 💠 · +{(s.darkMatter * 2).toFixed(0)}% income</div>
        {canPrestige(s) ? (
          <>
            <div class="unlock-reason">
              {s.rank >= 25 ? `Unlocked at Rank ${s.rank}.` : `Unlocked early via ₡10M+ lifetime earnings (still Rank ${s.rank}).`}
            </div>
            <PrestigeButton />
          </>
        ) : (
          <div class="empty-hint">Reach Rank 25 or ₡10M lifetime earnings to unlock.</div>
        )}
      </div>
      <div class="card">
        <div class="card-header"><span class="ch-icon">💠</span>RELICS</div>
        {RELICS.map((r) => {
          const level = s.relics[r.id] || 0;
          const maxLvl = relicMaxLevel(r);
          const maxed = maxLvl != null && level >= maxLvl;
          const cost = maxed ? 0 : relicCost(r, level);
          return (
            <div key={r.id} class="relic-row">
              <div class="r-icon">{r.icon}</div>
              <div class="r-info">

                <div class="r-effect">{r.effectLabel}</div>
              </div>
              <button class="btn btn-primary" disabled={maxed || s.darkMatter < cost} onClick={() => buyRelic(r.id)}>
                {maxed ? 'MAXED' : `${formatNum(cost)} 💠`}
              </button>
```

```tsx
  let best: { id: string; qty: number } | null = null;
```
